Always 1 — the button is never registering. This is a physical wiring issue, not a code issue. The code is correct.

Things to check on the hardware:

Pi Pin numbering — make sure you're using Board pin 12 (physical pin, counting from the corner), not GPIO numbering. Pin 12 is the 6th pin on the right column of the 40-pin header.

GND — Pin 6 is physical pin 6 (3rd pin on the right column). Any GND pin works though (pins 6, 9, 14, 20, 25, 30, 34, 39).

Button orientation — a 4-leg tactile button has two pairs of internally connected legs. Make sure the two wires go to opposite diagonal corners (or use a multimeter to find which legs are connected when pressed).

Try shorting GPIO18 to GND by hand — touch a wire from physical pin 12 to any GND pin. If the monitor prints 0, wiring is the issue. If still 1, there may be a bad GPIO pin.

Once you confirm the button reads 0 when pressed, the software will work immediately — the button detection code is correct.

Most likely cause: 4-leg tactile button — you need to connect across the correct pair of legs. A standard 4-leg button has legs A1-A2 (always connected) and B1-B2 (always connected). If both your wires land on the A side, they're always shorted together and the pin reads whatever the wire drives.
